2java类库中的LocalDate类

2.1标准的Java类库包含了一个用来表示时间点的Date类，一个用日历表示法表示日期的LocalDate类。

不要使用构造器来构造LocalDate类的对象，实际上应该使用静态工厂方法，它会代表你调用构造器，构造一个新的对象，表示构造这个对象时的日期。

//LocalDate类要用静态工厂方法构造对象
System.out.println(new Date().toString());//输出Sun Jun 21 09:55:26 CST 2020
System.out.println(LocalDate.now());//2020-06-21

2.2

//可以提供年月日，构造一个特定日期的对象
System.out.println(LocalDate.of(2000, 3, 4));//输出2000-03-04
//将构造的对象保存在对象变量里，用LocalDate的对象的方法getYear,getMonthValue,getDayOfMonth分别得到年月日
LocalDate a=LocalDate.of(2001,03,04);
System.out.println(a);//输出2001-03-04
int year=a.getYear();
int month=a.getMonthValue();
int day=a.getDayOfMonth();
System.out.println(month);//输出3

2.3

//plusDays属于访问器方法
LocalDate a = LocalDate.of(2000, 3, 4);
LocalDate b = a.plusDays(2);//对象a调用plusDays方法构造了新的对象，日期在原来对象上加两天，原来对象不变
System.out.println(a);//2000-03-04，对象a调用plusDays方法后对象a并未改变
System.out.println(b);//2000-03-06
System.out.println(a == b);//输出false,表示a,b引用的是两个不同位置的对象

3更改器方法和访问器方法

更改器方法（修改对象的方法）：调用更改器方法时，原来对象的状态（内容）会发生改变。

构造器方法：只访问对象而不修改对象的方法称为访问器方法，如：2.2中的getYear，getMonthValue，getDayOfMonth方法；

2.3中的plusDays方法。

LocalDate date = LocalDate.now();
System.out.println(date);//输出2020-06-21
int month = date.getMonthValue();
int today = date.getDayOfMonth();
date = date.minusDays(today - 1);//将date设为本月第一天
System.out.println(date);//输出2020-06-01

二、用户自定义类

接下来将仔细分析自己创建的Employee类，一个简单的工资管理系统源程序：

import java.time.LocalDate;

public class HelloHa {

Employee[] staff = new Employee[3];

​ e.raiseSalary(5);

class Employee
{
private String name;
private double salary;
private LocalDate hireDay;

public Employee(String n, double s, int year, int month, int day)
{
name = n;
salary = s;
hireDay = LocalDate.of(year, month, day);
}

public String getName()
{
return name;
}

public double getSalary()
{

return salary;
}

public LocalDate getHireDay()
{
return hireDay;
}

public void raiseSalary(double byPercent)
{
double raise = salary * byPercent / 100;
salary += raise;

}

1Employee类包含一个构造器和四个方法：

public Employee(String n,double s,int year,int month,int day)

public String grtName()

public double getSalary()

public LocalDate getHireDay()

public void raiseSalary(double byPercent)

这个类的所有方法都被白标记为public，关键字public意味着任何类的任何方法都可以调用这些方法（共有四种访问级别，在本章和下一章介绍）。

2Employee类的实例中有三个实例字段用来存放将要操作的数据：

private String name;

private double salary;

private LocalDate hierDay;

关键字private确保只有Employee类自身的方法才能够访问这些实例字段，而其他类的方法不能够读写这些字段。

注意：可以实用public标记实例字段，但这是一种很不好的做法。public数据字段允许程序中的任何方法对其进行读取和修改，这就完全破坏了封装。任何类的方法都可以修改public字段。因此强烈建议将实例字段标记为private。

3从构造器开始

3.1构造器与类同名，在构造Employee类的对象时，构造器会运行，从而将实例字段初始化为所希望的初始状态。如：

new Employee(“kkk”,100,2000,3,4);

会把实例字段设置为：

name=”kkk”;

salary=100;

hireDay=LocalDate.of(2000,3,4);

3.2构造器和其他方法有一个重要的不同：构造器总是结合new运算符来调用，不能对一个已经存在的对象调用构造器来达到设置实例字段的目的，如：

Employee a=new Employee(“kkk”,100,2000,3,4);

a. Employee(“aaa”,200,2001,5,6);//编译出错

3.3本章稍后将更详细介绍有关构造器的内容，现在只需记住：

(1)构造器与类同名

(2)每个类可以有一个及以上构造器

(3)构造器可以有0个，1个或多个参数

(4)构造器没有返回值

(5)构造器总是伴随着new操作符一起调用

3.4注意：不要在在构造器中定义与实例字段同名的局部变量，下面这个程序的构造器里声明了和实例字段相同的变量s,变量s在构造器内部访问时会将实例字段中s的值覆盖住，而在其他方法中访问变量s的值仍为数据字段中定义的值。

public class HelloHa {

​ public static void main(String[] args) {

​ A a=new A();//输出aaa
​ System.out.println(a.gets());//输出sss

}

}

class A{
String s=”sss”;
public A(){
String s=”aaa”;
System.out.println(s);
}
public String gets(){
return s;
}

}

这种错误很难检查出来，因此必须注意在所有的方法中都不要使用与实例字段同名的变量。

4用var声明局部变量

如果从变量的初始值就可以推断出它的类型，那么可以用var关键字声明局部变量，而无须指定类型，如用：

var harry=new Employee(“df”,4,4,4,4);

代替：Employee harry=new Employee(“df”,4,4,4,4);

从现在开始，倘若无须了解任何API就能从等号右边明显看出类型，在这种情况下都将使用var表示法，不过我们不会对数值类型使用var,因为要当心如0.0和0.0L之间的区别。

注意var关键字只能用于方法中的局部变量。参数和字段的类型必须声明。

5使用null引用

已经学过：一个对象变量包含一个对象的引用，或者包含一个特殊值null，null表示没有引用任何对象。

使用null时要非常小心：如果对null值应用一个方法，会产生一个NullPointException异常，如果程序没有捕获异常，程序就会终止。正常情况下，程序并不捕获这些异常，而是依赖于程序员从一开始就不要带来异常。

LocalDate bir=null;

Sreing s=bir.toString();//产生NullPointException异常
